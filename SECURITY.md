# Security Policy

感谢你帮助改进这个项目的安全性。

## 支持范围

当前优先支持的版本线：

- `0.1.x` 开发中版本
- `main` 当前主线

## 报告安全问题

请不要在公开 `Issue` 中直接披露未修复的安全问题。

推荐方式：

- 通过私有渠道联系仓库维护者
- 在报告中尽量包含复现步骤、影响范围、触发条件和建议修复方向

如果你暂时没有专门的安全邮箱，建议在仓库设置里补充一个专用安全联系方式。

## 我们关注的风险

- 凭证、`API key`、令牌或私密配置泄露
- 不安全的发布或打包流程
- 任意文件写入、路径穿越、命令执行类问题
- 数据解析与配置加载中的输入校验缺失

## 披露原则

- 给维护者留出修复和验证时间
- 修复发布后再公开细节
- 在确认影响面前，不要在公共讨论区贴出可直接复现的攻击细节
